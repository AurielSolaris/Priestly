 PRIESTLY -- DEVLOG
 A from-scratch encrypted messaging protocol over WSS.

 2026-07-16

TRANSPORT DECISION
  Weighed SSH vs WS, then TCP vs WSS vs QUIC for the base transport.
  Landed on WSS: framing + firewall/proxy traversal + TLS, without the
  head-of-line and browser-reach downsides that would push us to QUIC before we
  have a measured need for it. QUIC/WebTransport parked as a future option.

CONSTRAINT: FROM SCRATCH, MINIMAL DEPS
  Rule set for the build: implement the protocol ourselves on the standard
  library; the only runtime dep is pydantic, kept strictly for typing /
  validation / rule enforcement. No `websockets` library -- RFC 6455 is
  hand-rolled. Dev tooling (pytest, pyinstaller) allowed since it serves
  testing and packaging.

TRANSPORT LAYER (transport/)
  + ws.py    RFC 6455 from scratch: opening handshake (SHA-1 accept token),
             frame encode/decode, client masking, control frames (ping/pong/
             close), fragmented-message reassembly.
  + tls.py   stdlib ssl contexts -- the "S" in WSS. server + client sides.
  + server.py  threaded WSS server (one thread per connection).
  + client.py  WSS client.
  Verified end to end: real TLS -> WS handshake -> framed JSON, HELLO/MESSAGE
  ACKed by the server.

PROTOCOL LAYER (protocol/)
  + packets.py  versioned Packet envelope wrapping a discriminated union of
                frames (Hello / Message / Ack / Error), keyed on `type`.
                dumps()/loads() are the only bridge to the transport layer.
  models.py (Handshake, Message) retained as-is.

TEST SUITE (tests/) -- NASA/SQLite philosophy, one file per feature
  + test_frame_codec.py   frame round-trips at every length boundary + masking.
  + test_handshake.py     server/client handshake, RFC accept vector, errors.
  + test_websocket.py     send/recv, fragments, ping->pong, close, idempotency.
  + test_packets.py       envelope validation, discrimination, version guard.
  + test_tls.py           context construction + verification modes.
  + test_integration.py   live TLS+WS server: ACKs, 500 KB payload, bad-CA
                          rejection, clean close.
  Bugs caught by the suite and fixed in ws.py:
    * close() marked the socket closed before sending the Close frame, so the
      frame was swallowed -- reordered to send first.
    * recv() after close raised a raw OSError (WinError 10038); now translated
      to ConnectionClosed.

CLI SPLIT + COMPILATION
  Split the combined CLI into two standalone entry points (one job each):
    + cli/server.py  -> priestly-server(.exe)
    + cli/client.py  -> priestly-client(.exe)
  PyInstaller --onefile builds wired into the Makefile. Rationale: the test
  bench is a single machine, so client and server must run as separate
  processes. Verified: the two compiled binaries talk to each other over WSS.

BUILD/TEST WORKFLOW
  + Makefile  targets: install, cert, test, cov, build[-server|-client],
              run-server, run-client, clean.

CRYPTO PRIMITIVES (crypto/)
  + sha1.py     SHA-1 from scratch (RFC 3174) + payload validity tag/verify.
                Tested against hashlib as an oracle across length boundaries.
  + huffman.py  Huffman compression from scratch; deterministic tree so the
                decoder rebuilds identical codes from transmitted frequencies.
                Tested by lossless round-trip incl. empty / single-symbol / all
                256 byte values / random.
  Not yet wired into the packet path (next step).

CONFIG FORMAT: JSONX (config.py, config.cfg)
  Introduced JSONX = JSONL + JSONC:
    - JSONL: one JSON object per line; records merge top-to-bottom, later keys
      win (easy layering/overrides).
    - JSONC: //, #, /* */ comments and trailing commas, stripped string-aware.
  ServerConfig (pydantic, extra=forbid) validates server_name / host / port.
  Wired into both entry points; CLI flags override the file. Verified:
  config-driven boot ([priestly-node-1] on 8765) and flag override
  ([override-node] on 9100).

HOUSEKEEPING
  + .gitignore  ignore dist/build, *.spec, pytest/coverage artifacts, and
                certs/ (private key -- never commit).
  + README.md   architecture, principles, quickstart, config, build.

HARDENING + FUZZING
  Hardened transport/ws.py read_frame per RFC 6455: reject reserved bits,
  unknown opcodes, fragmented control frames, and oversized (>125) control
  frames with ProtocolError.
  + test_malformed_frames.py  truncated streams -> ConnectionClosed; protocol
                              violations -> ProtocolError; recv() propagates both.
  + test_packet_fuzz.py       loads() contract: any input either returns a valid
                              round-trip-stable frame or raises ValueError --
                              never another exception, never hangs. Random,
                              mutation, schema, and deep-nesting fuzzing.
  + test_integration.py       added scripted full session flow:
                              connect -> HELLO -> MESSAGE -> ACK -> clean close.

SECOND SECURITY LAYER: HMAC-SHA256 (crypto/hmac_sha256.py)
  Added a keyed authenticity layer on top of the unkeyed SHA-1 integrity tag.
  Deliberately NOT hand-rolled -- a keyed MAC is security-critical, so it uses
  the stdlib's audited, constant-time hmac + hashlib.sha256 (still zero external
  deps). sign()/verify() with constant-time comparison.
  + test_hmac_sha256.py  canonical vector, stdlib oracle, tamper/wrong-key/
                         wrong-tag rejection.

STATUS
  311 tests passing. Transport + protocol + config + compiled binaries working
  end to end, with hardened frame handling and fuzz-tested packet parsing. Crypto
  layers ready: SHA-1 (integrity) + HMAC-SHA256 (authenticity) + Huffman
  (compression). Next: integrate crypto into the packet path (per-message
  compression + validity/auth tag) and build out the crypto handshake.

REFRAME: PRIESTLY IS NOW PROTOCOL + PRODUCT
  A protocol is useless without a demo that exercises it, so Priestly became a
  two-layer system: the protocol (transport/protocol/crypto) plus a product
  (central chat server, CLI client, browser UI). Clients always connect to a
  central server and avoid P2P; the server is a decrypting relay.

LAYER 3: COVENANT (Module 3)
  Added an anonymous PAKE authentication layer on top of SHA-1 (integrity) and
  HMAC-SHA256 (authenticity). Transmits zero identity metadata; proves shared-
  secret knowledge without sending the secret; forward-secret via ephemeral DH.
  + crypto/hkdf.py        HKDF-Expand (RFC 5869) over HMAC-SHA256.
  + crypto/covenant.py    RFC 3526 Group 14 (verified 2048-bit prime), password
                          -> group element, iterative modinv, confirm MACs,
                          derive_session_keys (WPA3-style directional keys).
  + protocol/covenant.py  3-message handshake driver (commit/challenge/confirm)
                          returning a ready session, or CovenantError on failure.
  + protocol/session.py   Sealed messages: Huffman + HMAC-SHA256 tag over
                          epoch|seq|direction|payload, strict anti-replay. A
                          1-byte compression marker (stored/huffman) keeps the
                          format cross-implementable with the browser.
  As-built fixes vs the spec: synchronous (not async) to match the threaded
  transport; kept the server's confirm_s inside AUTH_OK so MUTUAL auth actually
  holds (the spec dropped it); wired Huffman into the packet path at last.
  New frames: hello_ack, covenant_commit/challenge/confirm, auth_ok/fail,
  sealed, chat.

QOL: PRODUCT LAYER
  + config.py             password from PRIESTLY_PASSWD / PASSWD env (overrides
                          config.cfg); password_required signalling.
  + cli/server.py         central chat server: HELLO -> HELLO_ACK -> [Covenant]
                          -> relay loop. Open mode (plaintext chat) or protected
                          mode (sealed). Rewrote from the old echo server.
  + cli/relay.py          thread-safe ClientRegistry; re-seals per recipient.
  + cli/chat_client.py    reusable client flow (shared with the tests).
  + cli/client.py         --listen / --ui / --password; clean auth-failure UX.
  + cli/ui_server.py      tiny stdlib HTTP server for the browser UI.

TRANSPORT ROBUSTNESS + PLAIN-WS MODE
  Reported: worker threads dumping ConnectionAbortedError (WinError 10053)
  tracebacks when a peer drops during the handshake (browsers probe the port,
  check the cert, close abruptly). Fixes:
  ~ transport/ws.py       _read_http_headers now translates socket OSError ->
                          ConnectionClosed (was raw traceback), matching
                          _recv_exact.
  ~ transport/server.py   _serve_connection also catches OSError as a safety net,
                          logging "connection dropped" instead of crashing the
                          thread.
  Reported: browsers refuse the self-signed cert (CERTIFICATE_UNKNOWN), blocking
  local UI testing. Added an opt-in no-TLS mode -> plain ws:// for local dev.
  Confidentiality is delegated away, but Covenant auth + per-message HMAC still
  hold; wss:// stays the default.
  + transport/{server,client}.py  use_tls flag; plain socket when disabled.
  + cli/server.py, cli/client.py  --no-tls; scheme printed on startup.
  + cli/ui_server.py, ui/index.html  inject ws/wss scheme into the page.
  + test_no_tls.py  plain-ws HELLO, Covenant auth, sealed relay, wrong password.

DOCS
  ~ README.md, DOCS/product.md  documented --no-tls local testing and the two
    ways to run the browser UI (plain ws vs trusting the dev cert).

STATUS
  410 tests passing. Full stack works end to end: (TLS or plain) WebSocket ->
  typed packets -> SHA-1 / HMAC-SHA256 / Covenant. The central server relays
  between authenticated clients; CLI and browser both drive the full flow; the
  browser crypto is verified against Python. Transport tolerates abrupt peer
  drops, and a --no-tls mode unblocks local browser testing. Possible next steps:
  handshake-phase socket timeouts (so mismatched ws/wss modes fail fast instead
  of hanging), epoch rekeying at seq rollover, an optional AES-GCM E2EE module,
  and PyInstaller data-bundling of ui/ for a single-file UI binary.
